1. PROBLEM STATEMENT
--------------------------------------------------------------------------------
Users need a way to browse the web in a private, isolated environment that:
- Does not leave traces on their local machine
- Provides a fresh browser instance for each session
- Runs with strict resource limits (CPU, memory) so one session cannot consume all system resources
- Cleans up automatically when the session ends (no leftover containers or data)

Traditional browsers run on the host and share cookies, cache, and fingerprint. Running a browser inside a Docker container gives isolation: each session is a separate container that is removed when the user is done.


2. PROJECT OBJECTIVE
--------------------------------------------------------------------------------
The objective of this project is to build a "Remote Ephemeral Browser" that:

- Lets users start a private browser session from a web UI
- Runs the browser inside an isolated Docker container (Chromium + noVNC)
- Streams the remote desktop to the user's browser via noVNC so they can see and use the remote Chrome
- Applies resource limits (512MB RAM, 1 CPU, 2GB shared memory) per container
- Automatically removes the container when the user clicks "End Session"

The backend is a Node.js (Express) API that uses the Docker API (via dockerode) to create, start, and stop containers. The frontend is a React + Vite app that calls the API and opens the noVNC URL in a new tab.


3. STEPS TO BUILD DOCKER IMAGES
--------------------------------------------------------------------------------

Step 3.1 - Prerequisites
- Install Docker on your machine and ensure the Docker daemon is running.
- Install Node.js 18 or higher (for local development of frontend; the backend can run fully in Docker).

Step 3.2 - Pull the browser image (required for the app to create sessions)
The application does not build the browser image; it uses a pre-built image from Docker Hub. You must pull it once before using the app.

On Linux / Windows (x86):
  docker pull nkpro/chrome-novnc:latest

On Apple Silicon (M1/M2/M3):
  docker pull --platform linux/amd64 nkpro/chrome-novnc:latest

This image contains Chromium and noVNC so the remote browser can be viewed in your browser.

Step 3.3 - Navigate to the backend directory
  cd backend

(Assume you are at the project root, e.g. rebrowser/, so backend is rebrowser/backend.)

Step 3.4 - Build the backend Docker image
  docker build -t rebrowser-backend:latest .

Explanation:
- -t rebrowser-backend:latest gives the image the name "rebrowser-backend" and tag "latest".
- The dot (.) means use the current directory (backend) as the build context. The Dockerfile in backend/ is used automatically.

Step 3.5 - (Optional) Tag for Docker Hub
If you want to push to Docker Hub, tag the image with your username and repository name:
  docker tag rebrowser-backend:latest YOUR_DOCKERHUB_USERNAME/docker_browser:latest

Replace YOUR_DOCKERHUB_USERNAME with your actual Docker Hub username (e.g. ayushman88).

Step 3.6 - (Optional) Push to Docker Hub
  docker login
  docker push YOUR_DOCKERHUB_USERNAME/docker_browser:latest

Summary of what we built:
- We did NOT build the browser image; we pulled nkpro/chrome-novnc:latest.
- We built one Docker image: the backend API (rebrowser-backend:latest), which the app uses to create and manage browser containers.


4. STEPS TO RUN CONTAINERS
--------------------------------------------------------------------------------

Step 4.1 - Ensure the browser image is available
  docker images | grep chrome-novnc

If you do not see nkpro/chrome-novnc, run:
  docker pull nkpro/chrome-novnc:latest
  (or with --platform linux/amd64 on Apple Silicon)

Step 4.2 - Run the backend as a container
The backend must run with access to the Docker socket so it can create and stop browser containers.

  docker run -d --name rebrowser-api -v /var/run/docker.sock:/var/run/docker.sock -p 3001:3001 rebrowser-backend:latest

Explanation:
- -d runs the container in the background (detached).
- --name rebrowser-api gives the container a name for easy reference.
- -v /var/run/docker.sock:/var/run/docker.sock mounts the host Docker socket into the container so the backend can talk to the Docker daemon.
- -p 3001:3001 publishes port 3001 so you can access the API at http://localhost:3001.
- rebrowser-backend:latest is the image we built in Section 3.

Step 4.3 - Run the frontend (for full app experience)
The frontend is not in a container in this project; run it locally:

  cd frontend
  npm install
  npm run dev

Then open http://localhost:5173 in your browser.

Step 4.4 - Use the application (this starts "session" containers)
- In the browser, go to http://localhost:5173.
- Click "Start Private Browser".
- The backend will create and start a new container from nkpro/chrome-novnc:latest and return a noVNC URL. A new tab will open with the remote Chromium desktop.
- Use the remote browser as needed. When done, click "End Session". The backend will stop the container, and because the container was created with --rm (auto-remove), Docker will remove it.

So we run two types of containers:
1. The backend container (rebrowser-api): long-lived, one per deployment.
2. Browser session containers: one per "Start Private Browser", removed when "End Session" is clicked.


5. COMMANDS USED
--------------------------------------------------------------------------------

Pull browser image (one-time):
  docker pull nkpro/chrome-novnc:latest
  docker pull --platform linux/amd64 nkpro/chrome-novnc:latest   (Apple Silicon)

Build backend image:
  cd backend
  docker build -t rebrowser-backend:latest .

Tag for Docker Hub (optional):
  docker tag rebrowser-backend:latest YOUR_DOCKERHUB_USERNAME/docker_browser:latest

Push to Docker Hub (optional):
  docker login
  docker push YOUR_DOCKERHUB_USERNAME/docker_browser:latest

Run backend container:
  docker run -d --name rebrowser-api -v /var/run/docker.sock:/var/run/docker.sock -p 3001:3001 rebrowser-backend:latest

Useful Docker commands while testing:
  docker ps
  docker logs rebrowser-api
  docker stop rebrowser-api
  docker rm rebrowser-api

Run frontend (no container):
  cd frontend
  npm install
  npm run dev
